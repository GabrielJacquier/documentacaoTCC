
Neste capítulo, apresenta-se a construção de uma ontologia de aplicação, para a área de dominio de atividades para MEIs. O processo de desenvolvimento da ontologia, teve como base os passos sugeridos na Metodologia 101. Os sub-capítulos seguintes, apresentam as etapas realizadas para implementação da ontologia.

Escopo

Para realizar a definição do escopo, utilizou-se questões que permitiram melhor entendimento sobre o dominio. O conteúdo das questões, serviu de base para realização de todas as outras etapas do desenvolvimento da ontologia.

Que domínio se deseja cobrir com a ontologia?
	A Classificação Nacional de Atividades Econômicas (CNAE) permitidas para o Micro Empreendedor Individual (MEI) no Brasil.

Com que propósito será utilizada a ontologia?
	Disponibilizar uma estrutura extensível e reutilizavel para aplicações que desejam obter informações sobre atividades permitidas para MEIs.

Que informações a ontologia deve fornecer?
	Quais atividades são permitidas para uma CNAE
	Qual CNAE corresponde a uma ocupação
	Qual CNAE permitie determinada atividade

Quem vai utilizar a ontologia?
	Aplicações que necessitam de maior semântica na obtenção de informações sobre atividades permitidas para MEI, assim como a validação se determinadas atividades podem ser realizadas por um CNAE.


Reuso de outras ontologias

A avaliação do reuso de outras ontologias, é importante para a reutilização de conhecimento. Assim permiti-se a diminuição do trabalho manual e a agregação de conceitos já validados por outros desenvolvedores à ontologia em desenvolvimento. 

Realizou-se uma pesquisa em repositórios online como Knoodl, Schemapedia e no repositório do Protégé, porém não foram encontradas ontologias que poderiam ser reutilizadas para o domínio abordado no projeto.

Conceitualização da Solução

Com o conhecimento obtido no experimento realizado, buscou-se uma nova conceitualização, tendo como objetivo, solucionar o problema semântico encontrado na consulta as atividades.

Foi então proposto a decomposição do conceito de atividade. A proposta foi realizada através da observação da estrutura das CNAEs, afim de obter para estas, uma representação suficientemente genérica. Amostra

Para criação das instâncias, utilizou-se uma amostra das CNAEs permitidas para MEI. Esta decisão, foi tomada pelo motivo do tempo necessário, para realizar a modelagem de todas CNAEs. 

No processo de seleção das CNAEs, não foram utilizados métodos estatísticos. Preferiu-se para efeito de enriquecimento dos testes, a escolha selecionada das CNAEs. Também deu-se importância para a diversidade da amostra, desse modo, permitiu-se aos testes maior credibilidade, com relação a generalidade da ontologia.

Foi contabilizado aproximadamente 500 CNAEs como permitidas para MEI, decidiu-se utilizar uma amostra de aproximadamente 4%, totalizando 20 CNAEs. Para assegurar a diversidade proposta, foram escolhidas CNAEs de diferentes seções. A Figura 7 apresenta as seções de maior representatividade.
