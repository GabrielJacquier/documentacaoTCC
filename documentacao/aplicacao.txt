Escopo

Para realizar a definição do domínio e o escopo da ontologia, pode-se utilizar de algumas questões para qualidade do resultado atingido.

Que domínio se deseja cobrir com a ontologia?
* Atividades permitidas para o Micro Empreendedor Individual (MEI)

Com que propósito será utilizada a ontologia?
* Disponibilizar uma estrutura extensível e reutilizavel para aplicações que desejam obter informações sobre atividades permitidas para MEIs.

Que informações a ontologia deve fornecer?
* Quais atividades são permitidas por um CNAE
* Qual CNAE corresponde a uma ocupação
* Qual CNAE permitie determinada atividade

Quem vai utilizar a ontologia?
* Aplicações que necessitam de maior semântica na obtenção de informações sobre atividades permitidas para MEI, assim como a validação se determinadas atividades podem ser realizadas por um CNAE.


Reuso de outras ontologias

A avaliação do reuso de outras ontologias, é importante para a reutilização de conhecimento, permitindo a diminuição do trabalho manual e podendo assim, agregar conceitos já validados por outros desenvolvedores. Foi realizada a pesquisa em repositórios online como Knoodl, Schemapedia e no repositório do Protégé, porém não foram encontradas ontologias que poderiam ser reutilizadas para o domínio abordado no projeto.

Amostra

Para realização da criação das instâncias, foi tomada a decisão da utilização de uma amostra sob o total das CNAEs existentes, por motivo de esforço e tempo necessário para então modelagem de todas CNAEs. Desta forma, foi possível o desenvolvimento do projeto.

Apresentamos o processo realizado para a seleção das CNAEs. Não foram utilizados métodos estatísticos para a montagem da amostra, pois foi necessário a escolha das CNAEs com atividades similares para problemática dos exemplos práticos, necessitando assim, da escolha manual pelos desenvolvedores. Porém foi tomado o devido cuidado para a amostra possuir diversidade e representatividade aceitáveis.

Foi contabilizado aproximadamente quinhentas CNAEs como permitidas para MEI, decidiu-se por uma amostra de aproximadamente 4%, totalizando 20 CNAEs. O número de CNAEs escolhido foi o maior possível, em relação ao tamanho do esforço manual e tempo disponível para se modelar as CNAEs na ontologia. Para assegurar a diversidade pretendida na amostra, foi escolhido diferentes seções, que serviram para a seleção das CNAEs. Foi contabilizada a quantidade de CNAEs em cada seção, e eleitas as seções com maior representatividade. A figura X apresenta as seleções escolhidas.

FiguraAmostraSecaoEscolhida1

Tendo as seções de maior representatividade, foi dividido igualmente o número de CNAEs, para cada uma das seções. A divisão de CNAEs foi igual entre as seções, para que cada seção tivesse um número considerável de CNAEs, assim facilitando futuros exemplos práticos.

A escolha das CNAEs foi realizada sob as seções selecionadas, foram escolhidos um grupo de CNAEs por seção garantindo a diversidade, e dentro de cada grupo buscou-se selecionar CNAEs com atividades de domínio similares.
